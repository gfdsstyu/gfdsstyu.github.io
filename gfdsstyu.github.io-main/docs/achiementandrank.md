제시해주신 수치(총 문제 690개, 성실 유저 일 30~40개, 헤비 유저 일 50~100개)를 기반으로 계산해보면, 기존 업적 난이도가 확실히 낮게 설정되어 있었을 가능성이 큽니다.

**수험생의 패턴 분석:**
* **성실 유저(일 35개 기준):** 690문제를 **약 20일**이면 1회독 마칩니다. (너무 빠름)
* **헤비 유저(일 80개 기준):** **약 8~9일**이면 1회독을 끝냅니다. (업적 고갈 속도 매우 빠름)

따라서, **'공부 중후반부(회독수 늘리는 시기)'의 동기부여**를 위해서는 단순히 문제를 '푼다'는 개념을 넘어, **'버틴다(지속성)'와 '반복한다(숙련도)'**에 초점을 맞춘 랭크 확장 및 난이도 상향이 필수적입니다.

다음은 수정 제안입니다.

### 1. 랭크 시스템 확장 (티어 세분화)

기존 **브론즈/실버/골드**에서 **플래티넘/다이아몬드/마스터**까지 확장하여, 시험 직전까지 계속 도전할 목표를 제시해야 합니다.

* **브론즈/실버:** 진입 단계 (초심자 배려)
* **골드:** 성실한 유저의 안정권 (1회독 완료 시점)
* **플래티넘:** 합격권 진입 (N회독 시작)
* **다이아몬드/마스터:** 고인물/수석 합격 도전권 (시험 1~2달 전 '양치기' 단계)

### 2. 업적 요건 재조정 (구체적 수치 제안)

업적을 크게 **A. 누적 문제수(끈기)**, **B. 일일 처리량(폭발력)**, **C. 회독 및 숙련도(합격 가능성)** 세 가지 축으로 나누어 재설계했습니다.

#### A. [누적] '감사 기계' (총 풀이 문제 수)
*헤비 유저가 하루 100문제씩 푼다고 가정할 때, 최소 1~2달은 지속 가능한 목표여야 합니다.*

| 랭크 | 기존(예상) | **변경 제안 (Total)** | 달성 예상 소요 (헤비/성실) | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **브론즈** | 50 | **100** | 1일 / 3일 | 맛보기 완료 |
| **실버** | 200 | **300** | 4일 / 10일 | 절반 정도 학습 |
| **골드** | 500 | **700** | 1주 / 3주 | **1회독 완료 상징** |
| **플래티넘** | - | **2,000** | 3주 / 2달 | 3회독 달성 (기출 암기 시작) |
| **다이아** | - | **5,000** | 2달 / 5달 | **무한 회독의 경지** |
| **마스터** | - | **10,000** | - | 수험 기간 내내 푼 수준 |

> **💡 포인트:** 골드를 '1회독 완료' 시점(약 700문제)에 맞추고, 플래티넘부터는 '회독수'를 늘려야 달성 가능하도록 수치를 대폭 높였습니다.

#### B. [일일] '벼락치기의 신' (하루 최대 풀이 문제 수)
*시험 막판 스퍼트(일 97문제)를 고려하여 상한선을 높입니다.*

| 랭크 | 변경 제안 (1일 기준) | 의도 |
| :--- | :--- | :--- |
| **브론즈** | **20문제** | 가볍게 몸풀기 |
| **실버** | **40문제** | 성실 유저의 평균치 |
| **골드** | **60문제** | 헤비 유저의 평상시 |
| **플래티넘** | **80문제** | 시험 직전 전범위 모의고사급 |
| **다이아** | **120문제** | **"오늘 미쳤다" (일 100개 돌파 도전)** |

> **💡 포인트:** 다이아몬드 조건을 '100문제+알파'로 설정하여, 유저가 "오늘은 기록 한 번 세워보자" 하고 작정하고 달리게 만듭니다.

#### C. [연속] '성실함의 증명' (연속 학습 일수)
*중후반부 슬럼프 방지를 위해 가장 중요한 지표입니다.*

| 랭크 | 변경 제안 (Day) | 의도 |
| :--- | :--- | :--- |
| **브론즈** | **3일** | 작심삼일 돌파 |
| **실버** | **7일** | 일주일 루틴 형성 |
| **골드** | **21일** | 습관 형성의 법칙 (3주) |
| **플래티넘** | **50일** | 슬럼프 극복 구간 |
| **다이아** | **100일** | **D-100일 전사** |

### 3. 중후반부 동기부여를 위한 '신규 업적' 아이디어

단순 문제 풀이 외에, 수험생의 심리를 자극하는 특수 업적을 추가하면 재미요소가 됩니다.

1.  **[N회독 마스터] (회독 시스템 연동 시)**
    * 조건: 전체 문제 DB(690개)를 한 바퀴 다 돌 때마다 뱃지 부여.
    * `1회독(입문)` -> `3회독(숙련)` -> `5회독(통달)` -> `7회독(해탈)`
    * *효과: 문제 수가 줄어드는 후반부에 "내가 몇 바퀴 돌렸나"를 시각화해줌.*

2.  **[새벽의 감시자] (시간대 타겟팅)**
    * 조건: 밤 12시 ~ 새벽 4시 사이에 문제 풀기 or 아침 6시~7시 기상 퀴즈
    * *효과: 시험 직전 밤샘 공부하는 유저들에게 소소한 위로와 보상.*

3.  **[스피드 레이서] (타임 어택)**
    * 조건: 1분 안에 3문제 정답 맞히기 (혹은 특정 챕터 평균 풀이 시간 상위 10%)
    * *효과: 이미 내용을 다 외운 '고인물' 유저들이 지루하지 않게 속도 경쟁을 유도.*

4.  **[오답 지우개] (약점 보완)**
    * 조건: 한 번 틀렸던 문제를 다시 풀어서 연속 50회 정답
    * *효과: 아는 문제만 푸는 '양치기'를 방지하고 오답 노트를 유도.*

### 4. 밸런스 요약

* **초반:** 브론즈/실버 등급은 쉽게 달성하게 하여 재미를 붙여줍니다.
* **중반(골드):** 1회독 완료 시점에 맞춰 달성감을 줍니다.
* **후반(플래티넘~다이아):** 단순히 '열심히' 해서는 안 되고, **'헤비하게'** 몰아치거나 **'꾸준히 N회독'**을 해야만 달성 가능하도록 벽을 높입니다. 그래야 시험 전날까지 업적창이 꽉 차는 것을 방지할 수 있습니다.

**다음 단계 제안:**
혹시 현재 구현된 DB 구조상 **'회독 수(전체 문제를 몇 번 리셋했는지)'**를 카운트할 수 있는 로직이 있나요? 있다면 [회독 마스터] 업적을 메인으로 내세우는 것이 수험생 입장에서 가장 매력적일 것입니다.


네, \*\*'업적 포인트(Achievement Point, AP)'\*\*를 도입하여 유저의 \*\*노력(문제 풀이량)\*\*과 \*\*성실함(출석)\*\*을 시각적인 \*\*'계급(Rank)'\*\*으로 치환하는 기획안입니다.

단순히 뱃지만 모으는 것보다, 남들에게 보여지는 **닉네임 색상**과 **티어**가 바뀌면 경쟁심리와 소속감을 동시에 자극할 수 있습니다. 특히 스터디 그룹 기능이 있다면 이 효과는 배가 됩니다.

### [기획안] 감린이 랭크 시스템 & 비주얼 아이덴티티

#### 1\. 업적 포인트 (AP) 산정 기준

기존 업적의 난이도에 따라 포인트를 차등 지급합니다. (총점 10,000점 만점 스케일 가정)

  * **일반 업적 (브론즈급 난이도):** 개당 **10 \~ 30 AP**
      * *예: 첫 문제 풀이, 3일 연속 출석, 튜토리얼 완료*
  * **중급 업적 (골드급 난이도):** 개당 **50 \~ 100 AP**
      * *예: 누적 500문제, 1회독 완료, 2주 연속 출석*
  * **상급 업적 (다이아급 난이도):** 개당 **200 \~ 500 AP**
      * *예: 누적 3,000문제(N회독), 일일 100문제 돌파, 오답노트 클리어*
  * **히든/특수 업적:** 개당 **300 AP**
      * *예: 새벽 4시 풀이, 크리스마스 당일 접속 등*

#### 2\. 랭크 티어(Tier) 및 진급 조건

수험생의 학습 단계(회독 수준)와 랭크를 매칭하여, 랭크만 봐도 \*\*"이 사람은 이 정도 고였구나"\*\*를 알 수 있게 설계합니다.

| 랭크 (Tier) | 필요 AP (예상) | 학습 단계 의미 (Flavor Text) | 닉네임 컬러 (Design) |
| :--- | :--- | :--- | :--- |
| **UNRANKED** | 0 | 감사 입문 (이제 막 책 폄) | 기본 검정/회색 (`#333333`) |
| **BRONZE** | 100+ | 초심자 (적응 중) | 구리색 (`#CD7F32`) |
| **SILVER** | 500+ | 숙련자 (강의 완강 수준) | 메탈릭 은색 (`#C0C0C0`) |
| **GOLD** | 1,500+ | **1회독 완료 (실력자)** | 빛나는 금색 (`#FFD700`) |
| **PLATINUM** | 3,000+ | N회독 러너 (합격권 진입) | 형광 민트/청록 (`#00CED1`) |
| **DIAMOND** | 5,000+ | **감사 기계 (고인물)** | 영롱한 다이아 블루 (`#50bcdf`) |
| **MASTER** | 8,000+ | 수석 도전 (인간 계산기) | 붉은빛 보라/오로라 (`#9932CC`) |

#### 3\. UI/UX 디자인 디테일 (호버링 & 아이콘)

유저가 그룹 멤버 목록 등에서 마우스를 올렸을 때(Hover), 시각적 만족감을 극대화하는 연출입니다.

**A. 랭크 아이콘 (Icon)**

  * 단순한 방패 모양이나 보석 모양보다는 **'회계/감사' 컨셉**을 살짝 섞거나, 아주 직관적인 **게이밍 스타일**을 추천합니다.
      * *브론즈/실버/골드:* 심플한 메달 형태.
      * *플래티넘:* 메달에 날개가 달림.
      * *다이아:* 중앙에 거대한 보석 + 광채 효과.
      * *마스터:* 왕관(Crown) 형태.

**B. 닉네임 호버링 효과 (CSS Styling Idea)**
랭크가 올라갈수록 닉네임에 단순히 색만 입히는 게 아니라, **Text-Shadow(빛 번짐)** 효과나 **Animation**을 줍니다.

  * **Bronze \~ Silver:** 단색 적용. 깔끔함.
  * **Gold:** 약간의 광택(Gradient) + 옅은 노란 그림자.
    ```css
    .user-gold {
        color: #FFD700;
        text-shadow: 0px 0px 5px rgba(255, 215, 0, 0.5);
        font-weight: bold;
    }
    ```
  * **Platinum:** 시원한 청록색 + 은은한 오라.
  * **Diamond:** **'블링블링' 효과.** 텍스트 색상이 미세하게 흐르거나(Background-clip), 강한 푸른 빛 발산.
    ```css
    .user-diamond {
        background: linear-gradient(to right, #30cfd0 0%, #330867 100%);
        -webkit-background-clip: text;
        -webkit-text-fill-color: transparent;
        text-shadow: 0px 0px 10px rgba(48, 207, 208, 0.6); /* 빛번짐 */
    }
    ```
  * **Master:** **불타는 효과** 혹은 **보라색 오라**. "건드리면 안 될 것 같은 포스".

**C. 호버 카드 (Hover Card) 정보**
마우스를 올렸을 때 뜨는 미니 팝업창 구성안:

1.  **닉네임 (특수 컬러 적용됨)**
2.  **현재 랭크 아이콘 & 명칭 (예: DIAMOND II)**
3.  **대표 업적 뱃지 3개** (유저가 자랑하고 싶은 것 3개 픽)
4.  **현재 상태 메시지** (예: "오늘 97문제 째려버리는 중🔥")

#### 4\. 동기부여 로직 (심화)

  * **강등 시스템 (선택사항):**
      * 너무 가혹할 수 있으나, **'휴면 유저'** 처리를 위해 2주 이상 미접속 시 랭크 아이콘에 '녹슨' 효과를 주거나 티어를 일시적으로 비활성화(회색 처리)하여 복귀를 유도할 수 있습니다. (점수 깎는 건 비추천, 의욕 상실 우려)
  * **승급전 (재미 요소):**
      * 예: 골드 -\> 플래티넘으로 넘어가는 구간(2900점 -\> 3000점)에서는 \*\*"승급 미션: 하루에 50문제 풀기"\*\*를 달성해야만 티어가 바뀌도록 설정. 긴장감을 줍니다.

이 기획대로라면 유저들은 단순히 문제를 푸는 게 아니라, \*\*"나 닉네임 파란색(다이아) 만들고 싶어서 오늘 50개 더 푼다"\*\*는 심리가 발동하게 됩니다. 개발 난이도 대비 유저 리텐션 효과가 매우 큰 기획입니다.
