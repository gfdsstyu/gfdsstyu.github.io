# 한국 감사기준 (KAS) RAG 데이터셋 - 최종 버전

## 📋 개요

**한국 회계감사기준 2025 개정판**을 RAG(Retrieval-Augmented Generation) 시스템 용 으로 정확히 파싱한 JSON 형식 데이터셋입니다.

- **문서**: 2025년 개정된 회계감사기준 전문
- **형식**: JSON (UTF-8)
- **항목 수**: 545개 (Heading 3 기반)
- **기준서 수**: 36개 (200~1200)

---

## 🔧 파싱 규칙 (최종 개정)

### 핵심: Heading 3 기반 카운팅

**이전 방식의 문제점:**
- 레벨 0/1 넘번링 기반으로 **143개** (기준서 240) 등으로 과다 파싱
- "List Paragraph", "불릿목록A" 등 혼재된 스타일로 정확도 저하
- A문단이 A47까지 가는 등 실제 구조와 불일치

**개선된 방식:**
- **Heading 1**: 기준서 경계 (감사기준서 200, 210, ...)
- **Heading 2**: 섹션 구분 ("요구사항" / "적용 및 기타 설명자료")
- **✅ Heading 3**: 실제 문단 (1-14, A1-A14)
- **Heading 4+, 기타**: 모두 하위 콘텐츠로 병합

```
기준서 320 예시:
Heading 3 #1: "이 감사기준서의 범위" → 320-1
Heading 3 #2: "감사에서의 중요성" → 320-2
Heading 3 #3: "시행일" → 320-3
Heading 3 #4: "감사계획..." → 320-4
Heading 3 #5: "감사진행..." → 320-5
Heading 3 #6: "문서화" → 320-6
[Heading 2: "적용 및 기타 설명자료"]
Heading 3 #7: "중요성과 감사위험" → 320-A1
Heading 3 #8: "감사에서의 중요성" → 320-A2
Heading 3 #9: "감사계획..." → 320-A3
Heading 3 #10: "감사진행..." → 320-A4
```
