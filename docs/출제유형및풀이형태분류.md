# 개발 참조 문서: 회계감사 문제 출제 유형 및 풀이 형태 분류

## 1. 개요 (Overview)

본 문서는 회계감사 수험 문제의 **출제 유형(Question Types)**과 **풀이 형태(Answer Forms)**를 코드로 분류하여 정의한다. AI 모델이 문제를 생성하거나 풀이를 제공할 때, 이 분류 체계를 기준으로 답변의 구조와 깊이를 결정하도록 프롬프팅한다.

## 2. 코드 매핑 테이블 (Code Mapping)

프롬프트 입력 시 아래의 조합 코드를 사용한다 (예: `A2`, `C3`). 

| 구분 | 코드 | 명칭 | 비고 |
| --- | --- | --- | --- |
| **출제 유형** | **A** | 단순암기형 (Simple Memorization) | 기준서 내용 인출 |
|  | **B** | 암기응용형 (Applied Memorization) | 사례 내 기준서 식별 |
|  | **C** | 계산형 (Calculation) | 표본감사 등 산수 수준 |
|  | **D** | 논리형 (Logic) | 제도 취지, 공익 보호 논리 |
|  | **E** | 실무형 (Practical Case) | 내부통제 및 실증절차 분석 |
| **풀이 형태** | **1** | 문장 서술형 | 3줄 이내 완결 문장 |
|  | **2** | 단답형 | 핵심 키워드 나열 |
|  | **3** | OX 판단형 | 판단 및 근거 서술 |
|  | **4** | 다지선다형 | 선택 및 근거 제시 |

## 3. 상세 정의 및 프롬프팅 가이드

### 3.1. 출제 유형 (Question Types) - `What`

AI는 문제의 맥락을 파악하여 다음 5가지 유형 중 하나로 분류하거나, 해당 유형에 맞춰 답변을 생성해야 한다.

#### **A. 단순암기형 (Code: A)**

* 
**정의:** 가장 기초적인 유형으로, 단순히 기준서 내용을 묻는 문제 


* 
**특징:** 문제 출제 비중이 높고 대다수 수험생이 득점하는 중요한 유형 


* **전략 가이드:**
* 원문에 가깝게 암기하여 서술하는 것이 최선이다 


* 핵심 키워드 위주로 타당하게 문장을 구성해도 고득점이 가능하다 





#### **B. 암기응용형 (Code: B)**

* 
**정의:** 사례(상황)가 주어지지만, 본질은 암기한 기준서를 변형하거나 그대로 인출하는 유형 


* 
**특징:** 사례 속에 출제자가 의도한 '기준서'가 숨겨져 있으므로 이를 찾아내는 것이 핵심이다 


* **전략 가이드:**
* 상황 묘사에 현혹되지 말고 적용될 기준서 문단을 인출하도록 유도한다 


* 단순암기형(A)과 유사한 방식으로 답안을 작성하되, 사례에 맞춰 조금씩 변형한다 





#### **C. 계산형 (Code: C)**

* 
**정의:** 표본감사(13장), 중요성 등에서 매년 1문제 이상 출제되는 수치 계산 유형 


* **전략 가이드:**
* 수학이 아닌 '산수' 수준이며, 공식 암기 시 점수 획득이 용이하다 





#### **D. 논리형 (Code: D)**

* 
**정의:** 제도의 취지, 이유, 효익 등을 서술하는 유형 


* **전략 가이드:**
* 정답은 없으나 모범 답안은 존재한다 


* 
**Tip:** 답변 생성 시 논리가 막힐 경우, **'공익(Public Interest) 보호'** 논리를 기반으로 서술하도록 프롬프팅한다 





#### **E. 실무형 (내부통제 및 실증절차) (Code: E)**

* 
**정의:** 단순 암기로 해결 불가한 최고 난이도 유형으로, 사례 속 위험과 오류를 분석해야 한다 


* **전략 가이드:**
* 암기한 기준서가 직접적으로 사용되지 않으므로, 다양한 사례 학습이 필요하다 


* 실무 경험이 없는 경우, '암기응용형(B)'과 유사하게 접근하여 생소한 상황에 대처하도록 한다

### 3.2. 풀이 형태 (Answer Forms) - `How`

AI는 출력 형식을 다음 4가지 중 하나로 고정해야 한다.

#### **1. 문장 서술형 (Code: 1)**

* 
**형식:** 주어와 서술어를 갖춘 완결된 문장 


* 
**제약:** 전통적 방식으로 3줄 이내로 작성 



#### **2. 단답형 (Code: 2)**

* 
**형식:** 문장 불필요, **핵심 키워드(단어)**만 간결하게 나열 


* 
**Trigger:** 문제에서 '~를 제시하시오' 또는 '~를 기재하시오'라고 묻는 경우 



#### **3. OX 판단형 (Code: 3)**

* 
**형식:** O/X 판단 + **틀린 이유(기준서 근거)** 반박 서술 


* 
**중요성:** 최근(24년 기준) 30문제 이상 출제되는 트렌드로, 합불을 가르는 가장 중요한 유형이다 


* **전략 가이드:**
* 단순히 맞히는 것이 아니라 근거를 서술하는 것이 핵심이다 


* 암기형과 사례형을 모두 아우르는 복합적 접근이 필요하다 





#### **4. 다지선다형 (Code: 4)**

* 
**형식:** 5개 보기 중 선택 + 이유 서술 


* 
**전략 가이드:** 정확한 근거를 통해 선택지를 좁히는 논리가 포함되어야 한다

## 4. 프롬프트 예시 (Example Usage)

사용자가 문제 생성 또는 풀이를 요청할 때, 아래와 같이 코드를 조합하여 지시한다.

> **User Request:**
> "표본감사와 관련된 간단한 수치 계산 문제를 내줘. (유형코드: `C3`)"
> **System Interpretation:**
> * **Type C (계산형):** 표본감사 공식을 활용한 산수 문제 생성.
> * **Form 3 (OX 판단형):** 계산 결과가 맞는지 틀린지 제시하고, 그 근거(올바른 계산식)를 서술하게 함.
> 
> 

> **User Request:**
> "내부통제 사례에서 취약점을 찾는 문제를 만들어줘. 답은 키워드만 딱 줘. (유형코드: `E2`)"
> **System Interpretation:**
> * **Type E (실무형):** 복잡한 내부통제 시나리오 제시.
> * **Form 2 (단답형):** 긴 서술 없이 핵심 취약점 단어만 나열.
> 
> 

> 
> **Example Mapping:** 
> 
> 
> * 단순암기 + 단답형 = **"A2"**
> * 암기응용 + 문장서술 = **"B1"**
> * 논리형 + 다지선다 = **"D4"**
> 
>

## 5. 구현 현황 (Implementation Status)

### 2025-01-23 구현 완료 항목

#### 5.1 기출문제 모드 - 단원별 문제풀이 기능
- **단원별 풀이 버튼**: 연도 선택 화면에 "📚 단원별 풀이" 버튼 추가
- **단원 선택 화면**: 기준서 단원(1장~15장)별 문제 카드 표시
  - 각 단원별 문제 수, 총 배점, 응시 횟수 표시
  - 최고 점수 뱃지 표시
- **단원별 문제풀이 화면**: 해당 단원의 모든 연도 문제를 통합하여 풀이
  - 제한 시간: 문제당 6분
  - 자동 저장 기능
- **단원별 채점 결과**: AI 채점 후 상세 피드백 제공

#### 5.2 플로팅 바로가기 컨트롤
- **연도별 문제풀이**: 우측 플로팅 컨트롤로 문제 번호 바로가기
- **연도별 채점결과**: 문제별 점수 색상 표시 (녹색=90%↑, 노랑=50%↑, 빨강=50%↓)
- **단원별 문제풀이**: 물음별 바로가기 컨트롤 추가
- **UI 스타일**: 세로 리스트 형태, 왼쪽 border 색상으로 상태 표시

#### 5.3 문제별 메모 기능
- **메모 저장**: 각 문제별 개인 메모 작성/수정/삭제
- **저장 위치**: localStorage (`exam_question_memos` 키)
- **연도별/단원별 공유**: 같은 문제 ID면 메모 공유
- **채점 결과 화면**: 각 문제 해설 아래 노란색 메모 영역 표시
- **PDF 내보내기**: 메모 포함 옵션 추가

#### 5.4 연도 선택 화면 UI 개선
- **오답만 다시 풀기**: 버튼 → 텍스트 링크로 변경 (카드 높이 축소)
- **결과보기 버튼**: 📊 아이콘만 표시 (컴팩트)
